\LoginController

LoginController Controls everything that is authentication-related

Whenever a controller is created, we also

  1. initialize a session
  2. check if the user is not logged in anymore (session timeout) but has a cookie

showCaptcha()

showCaptcha()

Generate a captcha, write the characters into $_SESSION['captcha'] and returns a real image which will be used like this: <img src=".

....../login/showCaptcha" /> IMPORTANT: As this action is called via <img ...> AFTER the real application has finished executing (!), the SESSION["captcha"] has no content when the application is loaded. The SESSION["captcha"] gets filled at the moment the end-user requests the <img .. > Maybe refactor this sometime.
